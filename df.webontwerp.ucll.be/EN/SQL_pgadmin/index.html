Installation pgAdmin

In this chapter, you will install and configure pgadmin, server connections, … This involves a lot at the beginning. Lots of things can (and will) go wrong but don't worry: we are here to help.

Connecting to the database server

You can install database software on your own laptop. For larger applications, however, you work with a database server. In our course, we use a PostgreSQL server. You communicate with the server via a client. There are dozens of clients. In this course we will use two of them: pgAdmin and DBSchema. For the first classes pgAdmin will do fine.

When you ran through all of the steps successfully, you will have three connections to the database:

  • The connection via port 54321 remains the same every year. Here you will find a number of basic databases that can be used over several academic years can be used.
  • The second connection, via port 52324 is updated every year. This academic year is 23-24, hence the choice of this port. Via this connection you reach, among other things, the database of your group where you can create your own new schedules yourself.
  • Finally, there is a third connection that uses pooling: 62324 (also new every year, so next year 62425 ...). This connection gives access to exactly the same as via port 52324, but in a smarter way so that you will never run out of connections. In practice, you always use this pooling connection.
